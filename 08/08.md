## 笔记

偏差，方差，正则化，训练集测试集以及验证集的划分。模型的选择

若模型没有捕捉到数据的趋势，那么就会有偏差。如果忽略模型趋势，过于追求模型拟合度，则称为过拟合(对于数据噪声极其敏感)。

### 正则化

防止过拟合的方法之一为正则化

假设为线性回归增加正则化：

$min \ \ \frac{1}{2}\sum_{i=1}^{m}\|y^{(i)}-\theta^{T}x^{(i)}\|^{2}+\frac{\lambda}{2}\|\theta\|^{2}$

$\frac{\lambda}{2}\|\theta\|^{2}$称为正则化项，当我们增加$\lambda$相当于增加对系数的惩罚，如若其过大，将会导致系数全为0，等于0相当于过拟合。

对于逻辑回归：

其损失函数为：$max \  \ \ \sum_{i=1}^{n}log\ p(y^{(i)}|x^{(i)};\theta)-\lambda\|\theta\|^{2}$

支持向量机不会太过拟合。（其目标函数可以类似于正则化）

逻辑回归在解决文本分类处理往往会弱于朴素贝叶斯，会导致模型崩溃，但是带正则化的逻辑回归往往会优于朴素贝叶斯。

为什么不对每个惩罚系数，因为数量过大难以进行抉择。

进行中心化处理可以使得迭代更快。



假设$S=\lbrace(x^{(i)},y^{(i)})\rbrace_{i=1}^{m}$

则有$P(\theta|S)=\frac{P(S|\theta)P(\theta)}{P(S)}$

因为数据集确定所以等效于$arg\underset{\theta}max\ \ P(S|\theta)P(\theta)=\prod P(S|\theta)P(\theta)$

在初步假设中假定$\theta\sim N(0,\sigma^{2}I)$,先验概率(贝叶斯学派)

map estimation(最大后验估计)$max \ \ P(\theta|S)$

频率学派：极大似然估计(MLE)

### 数据集划分

数据集分为训练集，验证集，测试集。验证集去选择模型

普遍经验法则：70%训练，30%测试。60%训练，20%验证，20%测试。

简单保持交叉验证，测试集有时也称交叉验证集

K折交叉验证：

将原始数据分为K份不同的子集，使用K-1份训练模型，剩下一份作为验证。

比K折法更极端的方法为留一法，

添加特征法（类似于逐步回归模型）
